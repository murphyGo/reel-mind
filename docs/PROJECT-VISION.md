# AI-DLC Based Automated Development Template Project

## Purpose

This project is a template generation project that enables idea concretization based on AI-DLC and allows entire development to be carried out with Claude through simple skill execution.
It uses feedback loops for continuous improvement, not ending with a single development cycle.

Users simply:
1. Present their initial idea
2. Let Claude develop it
3. Oversee the results

With just an idea, any project can use this template to elaborate AI-DLC based specifications and immediately proceed with development.

---

## Three-Stage Automation

### Stage 0: Interactive Requirements Refinement (Claude-Powered)

Before generating specs, Claude engages in an interactive dialogue to deeply understand and enhance the user's idea:

**Analysis Phase:**
- Read and comprehend inception.md
- Identify the core problem being solved
- Understand user's intent beyond what's written
- Detect implicit assumptions and unstated requirements

**Improvement Suggestions:**
Claude proactively suggests improvements in these areas:

| Category | What Claude Analyzes |
|----------|---------------------|
| **Completeness** | Missing features, edge cases, error scenarios |
| **Clarity** | Ambiguous requirements, vague descriptions |
| **Feasibility** | Technical complexity, potential blockers |
| **Architecture** | Component structure, service boundaries |
| **Non-Functional** | Performance, security, scalability considerations |
| **User Experience** | User flows, interaction patterns |
| **Integration** | External dependencies, API contracts |

**Interactive Loop:**
```
┌─────────────────────────────────────────────────────────┐
│  1. Claude reads inception.md                           │
│  2. Claude presents analysis + improvement suggestions  │
│  3. User approves / provides feedback / asks questions  │
│  4. Claude refines understanding                        │
│  5. Repeat until user confirms "proceed"                │
│  6. Generate enhanced requirements.md                   │
└─────────────────────────────────────────────────────────┘
```

**Output:**
- `docs/requirements.md` - Enhanced, structured requirements document
- `docs/refinement-log.md` - Record of the refinement conversation

### Stage 1: AI-DLC Inception (Spec Generation)
- Use enhanced requirements.md as input
- Generate vision.md and tech-env.md (AI-DLC inputs)
- Run AI-DLC workflow to produce aidlc-docs/

### Stage 2: Skill Generation
- Analyze generated aidlc-docs/
- Generate project-specific dev skills
- Include common skill templates (code-review, tech-debt, cross-check)

## Input

### Source Files
- `inception.md`: Document containing user's ideas and specific requirements

### Input Format: inception.md

Required sections:
- **Project Name**: Short identifier
- **Vision**: 1-2 sentence description of what to build
- **Core Features**: Bulleted list of must-have features
- **Tech Preferences** (optional): Language, framework, cloud preferences
- **Constraints** (optional): Budget, timeline, team size considerations

Example:
```markdown
# My Project

## Vision
A REST API for managing personal book collections

## Core Features
- User authentication (JWT)
- CRUD operations for books
- Search by title/author
- Export to CSV

## Tech Preferences
- Language: Go
- Database: PostgreSQL
- Cloud: AWS
```

## Interactive Refinement: Claude's Approach

### What Claude Analyzes

When reading inception.md, Claude should think like a senior architect:

**Understanding Intent:**
- What is the user really trying to achieve?
- What problem are they solving for their end users?
- What's the business context?

**Identifying Gaps:**
- What happens when things go wrong? (error handling)
- What about edge cases? (empty lists, concurrent access)
- What's not mentioned but essential? (auth, logging, monitoring)

**Challenging Assumptions:**
- Is this the right technical approach?
- Are there simpler alternatives?
- What are the scaling implications?

### Suggestion Format

Claude presents suggestions in a structured, actionable format:

```markdown
## Analysis of Your Idea

I've analyzed your inception.md. Here's my understanding and suggestions:

### What I Understand
[Summary of the project intent in Claude's own words]

### Suggestions for Improvement

#### 1. Completeness
**Missing: User authentication flow**
- Your API handles book data but doesn't specify how users are authenticated
- Suggestion: Add JWT-based auth with refresh tokens
- Questions: Should users be able to register, or is it invite-only?

#### 2. Edge Cases
**Not addressed: Concurrent book updates**
- What happens if two users edit the same book?
- Suggestion: Add optimistic locking with version numbers

#### 3. Non-Functional
**Consider: Rate limiting**
- Public APIs typically need protection from abuse
- Suggestion: Add rate limiting (100 req/min per user)

### Questions for You
1. Should deleted books be soft-deleted or permanently removed?
2. Do you need audit logging for compliance?
3. What's your expected data volume? (affects DB choice)

---
Ready to proceed with these enhancements, or would you like to discuss further?
```

### Dialogue Patterns

| User Response | Claude's Action |
|---------------|-----------------|
| "proceed" / "looks good" | Generate requirements.md and continue |
| "yes to all" | Apply all suggestions, generate requirements.md |
| "skip X" | Exclude suggestion X, proceed with others |
| Asks clarifying question | Explain in detail, then re-prompt |
| Provides additional context | Incorporate and re-analyze |
| "add feature Y" | Add to requirements, suggest related considerations |

## Feedback Loop Mechanism

The system supports continuous improvement through:

1. **Session Logs**: Every dev session creates a log for traceability
2. **TECH-DEBT Tracking**: Issues are captured and prioritized
3. **Cross-Check Reports**: Phase completion triggers compliance verification
4. **Plan Evolution**: development-plan.md updates as work progresses

This enables:
- Multiple development iterations
- Learning from previous sessions
- Accumulated context for Claude

## Success Criteria

The template is successful when:

- [ ] User can start with only a rough idea in inception.md
- [ ] Claude provides meaningful improvement suggestions (not generic)
- [ ] Interactive refinement produces better requirements than user's original input
- [ ] User feels their intent is understood, not just their words
- [ ] AI-DLC specs are auto-generated from enhanced requirements
- [ ] Project-specific dev skill is functional
- [ ] Common skills (code-review, tech-debt) work out of the box
- [ ] Feedback loop enables continuous iteration
- [ ] New developers can onboard by reading generated docs
