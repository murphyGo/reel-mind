# Example: Book Tracker

A complete walkthrough showing how AI-DLC Starter transforms a rough idea into a project-ready state.

## The Scenario

A developer has a rough idea: *"I want to build something that helps me track my reading habits."*

They use AI-DLC Starter to transform this into a fully-specified project.

## Stages

| File | Stage | What Happened |
|------|-------|---------------|
| [1-idea.md](1-idea.md) | After `/ideate` | Rough one-liner → structured IDEA.md |
| [2-requirements.md](2-requirements.md) | After `/init-project` Stage 0 | Enhanced requirements via dialogue |
| [3-aidlc-state.md](3-aidlc-state.md) | After `/init-project` Stage 1 | AI-DLC inception artifacts generated |
| [4-final-structure.md](4-final-structure.md) | After `/scaffold` + Stage 2 | Final project tree with skills |

## Key Observations

- The **initial idea was one sentence** — no tech stack, no features, no target users.
- `/ideate` added structure through **4 questions** across 2 rounds.
- Smart tech defaults kicked in because the user said "I don't know" for tech — Go + Chi + PostgreSQL was suggested.
- `/init-project` identified **3 missing concerns** (auth, error handling, data validation) and turned 3 vague features into 6 well-specified FRs.
- `/scaffold` generated 8 directories and 7 config files in one step — no code.
- Total interactive time: ~15 minutes.

## Viewing This Example

Each file shows the artifact as it would exist at that stage. The example is **not a real project directory** — it's a set of markdown snapshots intended as reference.

To see the full flow in action, run it yourself:

```
/start
```
